Carbon MCP Server

A Model Context Protocol server that exposes the Northwood portfolio's carbon data as callable tools. Any MCP-compatible agent — Claude Desktop, Claude Code, Cursor — can query emissions, run gap analyses, and simulate decarbonization scenarios without touching raw data.

What it is

Traditional dashboards serve pixels to humans. An MCP server serves structured capabilities to an LLM. The agent asks "which portcos are off-track for 2030?" — Claude picks the right tools, calls them in sequence, and writes the answer, tracing every number back to the data layer.

Tool catalog

Five tools, chosen to answer ~80% of real analyst questions without the agent ever touching raw CSV or JSON.

list_portcos()
All 10 portcos with sector, fund, revenue, EBITDA, facility count, data grade, and SBTi target.
→ ["meridian", "sterling", ...] sector, status, revenue_m, target_reduction_pct, ...
get_portco_emissions(portco, year?, scope?)
Scope 1 / Scope 2 emissions for a portco, in tCO2e. Optional year and scope filters.
→ get_portco_emissions( portco="meridian", year=2024) ← {"2024": {scope1: 750, scope2: 1850, total: 2600}}
gap_to_target(portco, target_year=2030)
Gap between current trajectory and SBTi-aligned reduction pathway. Returns verdict + numbers.
→ gap_to_target("harvest") ← {gap_pct: 31.4, on_track: false, verdict: "Materially off-track..."}
list_initiatives(portco?, status?)
Decarbonization levers with estimated reduction (tCO2e), capex ($k), category, and status.
→ list_initiatives( portco="sterling", status="planned") ← [3 initiatives with est_reduction, capex_k, ...]
simulate_reduction(portco, initiative_ids)
What-if: apply a subset of initiatives, return projected emissions, capex total, and gap to target.
→ simulate_reduction( portco="sterling", initiative_ids=[4, 5]) ← {projected_tco2e: 3960, meets_target: false, total_capex_k: 850}

Install & connect

The server is ~200 lines of Python using FastMCP. Clone the repo, then add it to your MCP client config.

Run locally

git clone https://github.com/trippborstel-hub/northwood-carbon-mcp.git cd northwood-carbon-mcp uv sync uv run server.py

Register with Claude Code / Claude Desktop

{ "mcpServers": { "northwood-carbon": { "command": "uv", "args": ["--directory", "/path/to/northwood-carbon-mcp", "run", "server.py"] } } }
Design note. The tool surface is deliberately small — 5 tools, each doing one clear thing. Good agent-app design rewards clarity of description over breadth of feature: if an agent can read a tool's name and one-sentence purpose and know exactly when to reach for it, it'll compose them correctly. Every extra tool adds noise to the agent's decision, not power.
