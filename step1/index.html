5 Key Decision Points Ahead

Decision 1: Keystone Scope — Week 3 Checkpoint

If Keystone's data doesn't materially improve by Week 3 (R-07 clinic utility data and R-13 therapist mileage don't arrive), we pivot: deeper quantitative analysis on Meridian and Sterling, lighter architecture-focused scope on Keystone. We'll flag this early — no surprises.

Decision 2: Add Harvest Ridge as Fourth Priority Portco?

Harvest Ridge scored 4/5 on data quality with a compelling sector story (food manufacturing, food waste, high-GWP refrigerants). Low marginal effort to include. This is Sarah's call — we've laid out the case in the Data Ask memo. Need answer by end of Week 2 to incorporate into Step 2 build.

Decision 3: CC Environment — Build or Wait?

If CC access isn't available by the end of Week 2, we recommend building the prototype in Kith's local environment and migrating later. This adds a migration step but prevents the system build from falling behind schedule. We'll raise this with Sarah if R-01 isn't resolved by April 3.

Decision 4: Scope 3 Depth — Screening vs. Quantification

The proposal scopes Scope 3 as a "screening" — identifying which categories are material. The data environment has zero Scope 3 inputs. If Tier 4 data (waste manifests, procurement spend, travel data) arrives, we can upgrade from screening to partial quantification for the most material categories. If it doesn't, we screen using industry benchmarks and flag specific categories for future data collection. Either approach is defensible — the question is how much client effort to invest in Scope 3 data for this engagement vs. building the collection system for next year.
