# Quarterly Board Presentation Template

**Northwood Capital Partners**
Portfolio Company Board of Directors Meeting

---

## Document Standards

| Parameter | Specification |
|-----------|--------------|
| Format | Slide deck (PowerPoint/Google Slides) or structured markdown |
| Slide dimensions | 1920 x 1080 (widescreen 16:9) |
| Target length | 25-35 slides (excluding appendix) |
| Distribution | Board members + observers, circulated 3 business days before meeting |
| Financial basis | Adjusted EBITDA, consistent with IC memo conventions |
| Comparison periods | Current quarter vs. budget vs. prior year quarter vs. prior year YTD |
| Rounding | Currency: $X.XM; Percentages: X.X%; Headcount: exact integer |
| Confidentiality | "CONFIDENTIAL — Northwood Capital Partners" footer on every slide |

## Section 1: Cover Page

**Purpose:** Establish the document as a formal board communication. Set the context — which company, which quarter, which meeting.

**Required elements:**

```
[COMPANY LOGO]

[Company Legal Name]
Board of Directors Meeting

[Quarter] [Year] Operating Review
[Full Date of Board Meeting]

CONFIDENTIAL
Prepared for the Board of Directors
Northwood Capital Partners — Sponsor Representative
```

**Formatting guidance:**

- Company logo prominently placed (top center or top left)
- Clean layout — no clutter, no data on the cover
- Include the Northwood confidential legend at the bottom
- Board meeting date (not the reporting period date) as the primary date
- Include meeting logistics: time, location (or video conference link), expected attendees

**Board attendee roster (include on cover or second slide):**

```
Board Members:
  [Name], [Title] — [Affiliation] (Chair)
  [Name], [Title] — [Affiliation]
  [Name], [Title] — [Affiliation]
  [Northwood Deal Lead], [Title] — Northwood Capital

  [Operating Partner], Operating Partner — Northwood Capital
  [Name], [Title] — [Company]

Management Presenters:
  [CEO Name], Chief Executive Officer
  [CFO Name], Chief Financial Officer
  [COO Name], Chief Operating Officer
```

## Section 2: Executive Summary / CEO Letter

**Purpose:** The CEO's direct communication to the board. This is not a data dump — it is a narrative assessment of business performance, key wins, key concerns, and near-term outlook. The board should leave this section understanding whether the business is on track, ahead, or behind, and why.

**Target length:** 2-3 slides

**Required content:**

- **Quarter in a sentence:** One-line characterization of the quarter (e.g., "Strong top-line quarter with margin pressure from wage inflation in the staffing segment")
- **2-3 key accomplishments:**
  - Specific, measurable achievements
  - Link to value creation plan initiatives where applicable
  - Examples: "Closed [Add-on] acquisition, adding $X.XM revenue," "Renewed [Customer] contract for 3 years at 5% rate increase," "Reduced DSO by X days through billing automation"
- **2-3 key concerns or challenges:**
  - Honest assessment of what is not working
  - Include management's response or corrective action
  - Examples: "Technician turnover increased to X% — implemented retention bonus program," "Q4 revenue $X.XM below budget due to project delays — $X.XM pushed to Q1"
- **Outlook for next quarter and balance of year:**
  - Revenue and EBITDA trajectory relative to budget
  - Pipeline and backlog visibility
  - Key milestones or inflection points ahead
  - Risks to watch

**Formatting guidance:**

- Write in the CEO's voice — first person, direct, not overly formal
- Use bullet points, not paragraphs. Board members scan, they do not read essays.
- Bold the key data points within bullets
- If a slide has more than 6 bullets, it has too many bullets

**Common mistakes to avoid:**

- CEO letters that are only good news. The board expects candor. If something is off track, say so before they ask.
- Vague statements like "solid quarter" without quantification. Every characterization needs a number.
- Failing to connect performance to the value creation plan. The board approved a plan — report against it.
- Outlook section that is just "we expect continued growth." Specify what growth, where, and what could derail it.

## Section 3: Financial Summary

**Purpose:** Present a clear, variance-driven financial overview. The board needs to understand actual performance relative to budget and prior year, with explanations for material variances.

**Target length:** 3-4 slides

**Required content:**

**Slide 3a: P&L Summary (quarterly)**

| Line Item | Q[X] Actual | Q[X] Budget | Var ($) | Var (%) | PY Q[X] | YoY ($) | YoY (%) |
|-----------|-------------|-------------|---------|---------|---------|---------|---------|
| Revenue | | | | | | | |
| COGS | | | | | | | |
| Gross Profit | | | | | | | |
| Gross Margin % | | | | | | | |
| SG&A | | | | | | | |
| Adjusted EBITDA | | | | | | | |
| EBITDA Margin % | | | | | | | |

**Slide 3b: P&L Summary (YTD)**

Same format as quarterly, YTD actual vs. YTD budget vs. PY YTD.

**Slide 3c: Key Variance Explanations**

For every line item with a variance >5% vs. budget or >10% vs. prior year:

```
Revenue: $[X.X]M vs. $[X.X]M budget ([+/-X.X]%)
  - [Driver 1]: $[X.X]M impact — [explanation]
  - [Driver 2]: $[X.X]M impact — [explanation]
  - [Driver 3]: $[X.X]M impact — [explanation]
```

**Slide 3d: Trailing 12-Month Trend (optional but recommended)**

- LTM revenue and EBITDA trend chart (bar + line)
- Shows momentum and seasonality
- Include LTM margin on secondary axis

**Formatting guidance:**

- Use consistent color coding: green for favorable variance, red for unfavorable, black for neutral
- Material variances (>5% of budget) should have narrative explanation — never leave a red number unexplained
- Favorable and unfavorable should be defined from the shareholder's perspective (higher revenue = favorable, higher SG&A = unfavorable)
- Always show both dollar and percentage variances

**Common mistakes to avoid:**

- Presenting financials without variance explanations. Numbers without narrative are useless to the board.
- Mixing reported and adjusted figures without clear labeling. If you present Adjusted EBITDA, show the bridge from reported.
- Omitting YTD. A bad quarter can be masked by a good YTD or vice versa. Show both.
- Revenue variance explanations that only say "timing." If revenue shifted between quarters, explain what specifically shifted and whether it is truly timing or a demand signal.

## Section 4: KPI Dashboard

**Purpose:** Provide a at-a-glance view of operational and financial health through 6-10 key performance indicators. The dashboard should enable rapid pattern recognition — what is on track, what is not, and what is trending.

**Target length:** 1-2 slides

**Required content:**

Select 6-10 KPIs from the categories below. Every KPI must include:

| KPI | Current | Prior Qtr | YoY | Target | RAG |
|-----|---------|-----------|-----|--------|-----|
| [Metric] | [Value] | [Value] | [Value] | [Value] | [R/A/G] |

**RAG status definitions:**

- **Green:** On or above target
- **Amber:** Within 5-10% of target, or trending unfavorably
- **Red:** More than 10% below target, or critical threshold breached

**Standard KPI categories (select relevant subset per portfolio company):**

Financial:
- Revenue (LTM and quarterly)
- Adjusted EBITDA and margin
- Free cash flow
- Net leverage ratio

Customer/Revenue Quality:
- Customer count / patient count / clinic count
- Customer retention rate / net revenue retention (NRR)
- Revenue per customer / average contract value
- Pipeline / bookings / backlog
- Customer concentration (top 1, top 5)

Operational:
- Headcount (total, by category)
- Employee turnover / retention rate
- Utilization rate (for services businesses)
- On-time delivery / quality metrics
- Safety (recordable incident rate — for industrial/field services)

Growth:
- New customer wins
- New locations opened / de novo clinics
- Add-on acquisition pipeline status

**Formatting guidance:**

- Dashboard should fit on 1 slide if possible (2 maximum)
- Use visual indicators (arrows, color dots) for trend direction
- Include sparkline or mini-charts for each KPI showing 4-8 quarter trend
- RAG colors should be immediately visible — do not require the reader to interpret numbers
- Order KPIs by importance, not alphabetically

**Common mistakes to avoid:**

- Too many KPIs. If everything is a KPI, nothing is. Pick the 6-10 that actually drive decisions.
- KPIs without targets. A metric without a target is information, not a KPI.
- RAG ratings that never turn red. If every metric is green for 4 quarters, the targets are too easy or the ratings are dishonest.
- Operational KPIs disconnected from financial outcomes. Every operational KPI should have a known linkage to revenue, margin, or risk.

## Section 5: Revenue Detail

**Purpose:** Provide granular visibility into revenue composition, trends, and forward indicators. The board should understand where growth is coming from, where it is slowing, and what the pipeline looks like.

**Target length:** 2-3 slides

**Required content:**

**Slide 5a: Revenue by Segment / Vertical / Geography**

- Revenue by business line or service category (if multi-line)
- Revenue by customer vertical or industry
- Revenue by geography (if multi-region)
- Show current quarter, YTD, LTM, and budget comparison for each cut
- Highlight fastest-growing and fastest-declining segments

**Slide 5b: Revenue Quality Metrics**

- Recurring vs. non-recurring revenue split (with trend)
- Contract renewal rates and pricing trends
- Net revenue retention (NRR) for recurring businesses
- Average contract value and term
- Cohort analysis (where applicable)

**Slide 5c: Pipeline / Bookings / Backlog**

- New business pipeline (by stage, expected close, value)
- Bookings vs. prior period
- Backlog (contracted but unrecognized revenue)
- Win rates and competitive dynamics
- Key deals won and lost (with brief narrative)

**Formatting guidance:**

- Use waterfall charts to show revenue bridges (prior year -> growth drivers -> current year)
- Pie charts for mix (but only if the mix has changed — otherwise use a table)
- Pipeline should be presented by stage with weighted and unweighted values
- Forward-looking revenue indicators should be clearly distinguished from actual results

**Common mistakes to avoid:**

- Revenue mix analysis without commentary on mix shift. If a lower-margin segment is growing faster, that is a margin risk — flag it.
- Pipeline data without conversion rates. A $50M pipeline with 10% close rate is $5M. A $20M pipeline at 50% is $10M. Unweighted pipeline numbers are misleading.
- Omitting customer losses. New customer wins get reported; customer churn often does not. Always report both sides.

## Section 6: Operations Update

**Purpose:** Report on the operational health and execution of the business — people, process, quality, and safety. This section is particularly important for labor-intensive businesses (healthcare, staffing, field services, manufacturing).

**Target length:** 2-3 slides

**Required content:**

**Slide 6a: People / Headcount**

- Current headcount by department / function / region
- Hiring vs. plan (open positions, time-to-fill)
- Turnover rate (voluntary and involuntary) vs. target
- Key hires and departures in the quarter
- Compensation and benefits cost trends

**Slide 6b: Operational Metrics**

Select metrics relevant to the business type:

- *Services:* Utilization rate, billable hours, bill rate trends, project backlog
- *Healthcare:* Patient volume, visits per clinic, payor mix, claim processing time
- *Staffing:* Fill rate, temp workers on assignment, bill rate vs. pay rate spread, workers comp loss ratio
- *Manufacturing/Industrial:* Production volume, quality yield, on-time delivery, machine utilization, safety (TRIR)
- *IT/Technology:* Ticket volume, SLA compliance, mean time to resolution, platform uptime

**Slide 6c: Facility / Infrastructure (if applicable)**

- Location count and status (open, under construction, planned)
- Capacity utilization by location
- Capital projects in progress (budget, timeline, status)
- Lease expirations in next 12-24 months

**Formatting guidance:**

- Headcount should always be shown as a bridge: beginning + hires - departures = ending
- Turnover should be benchmarked against industry averages
- Operational metrics should include trend arrows and RAG status
- Safety metrics (if applicable) should be prominently placed — this is a board-level concern

**Common mistakes to avoid:**

- Headcount without turnover context. Growing headcount can mask high turnover. Report both.
- Operational metrics without benchmarks. Is 72% utilization good? Depends on the industry. Provide context.
- Omitting safety and compliance. For industrial, healthcare, and field services portfolio companies, these are not optional slides — they represent legal and reputational risk.

## Section 7: Strategic Initiatives

**Purpose:** Report progress against the value creation plan approved at acquisition. This section connects the quarterly operating results to the long-term investment thesis.

**Target length:** 2-3 slides

**Required content:**

**Slide 7a: Value Creation Plan Scorecard**

| Initiative | Owner | Target | Status | Progress | Timeline | EBITDA Impact |
|-----------|-------|--------|--------|----------|----------|--------------|
| [Initiative 1] | [Name] | [Target] | [RAG] | [Description] | [On time / Delayed] | $[X.X]M |
| [Initiative 2] | [Name] | [Target] | [RAG] | [Description] | [On time / Delayed] | $[X.X]M |
| ... | | | | | | |

**Slide 7b: Add-on Acquisition Pipeline**

| Target | Sector | Revenue | EBITDA | EV | Multiple | Stage | Expected Close | Lead |
|--------|--------|---------|--------|----|---------| ------|---------------|------|
| [Name] | [Sector] | $[X]M | $[X]M | $[X]M | [X.X]x | [Stage] | [Date] | [Name] |

- Pipeline summary: total targets identified, under NDA, in DD, LOI submitted
- Completed add-ons YTD: integration status and performance vs. underwriting
- Available funding: revolver capacity, incremental term loan availability, co-invest

**Slide 7c: Key Strategic Projects**

For each major project (system implementation, facility expansion, new market entry):

```
Project: [Name]
Owner: [Name]
Budget: $[X.X]M (spent to date: $[X.X]M)
Timeline: [Start] to [Expected completion] — [On track / X weeks behind]
Key milestones this quarter: [Completed milestones]
Next quarter milestones: [Upcoming milestones]
Risks: [Key risks to delivery]
```

**Formatting guidance:**

- Value creation plan scorecard should be a single-slide summary that the board can scan in 30 seconds
- Use RAG status consistently with KPI dashboard definitions
- Add-on pipeline should distinguish between "watching" and "actively pursuing"
- Each initiative should have a named owner — if nobody owns it, it is not real

**Common mistakes to avoid:**

- Value creation plans that never update. If the plan from 2 years ago is still showing the same initiatives with the same status, the plan is stale.
- Add-on pipelines with no movement. If you have shown the same 5 targets for 3 quarters with no progress, explain why and whether the strategy needs revision.
- Strategic initiatives without financial impact estimates. The board approved a value creation plan with specific EBITDA targets. Report against them.

## Section 8: Risks & Issues

**Purpose:** Maintain a living risk register and alert the board to new, escalating, or de-escalated risks. This is a forward-looking section — the board wants to know what could go wrong, not just what has gone wrong.

**Target length:** 1-2 slides

**Required content:**

**Risk Register Update:**

| # | Risk | Category | Severity | Likelihood | Priority | Change | Mitigant | Owner |
|---|------|----------|----------|------------|----------|--------|----------|-------|
| 1 | [Risk] | [Cat] | [H/M/L] | [H/M/L] | [Score] | [New/Up/Down/Unchanged] | [Mitigant] | [Name] |
| 2 | [Risk] | [Cat] | [H/M/L] | [H/M/L] | [Score] | [New/Up/Down/Unchanged] | [Mitigant] | [Name] |

- **New risks** identified this quarter (highlight with "NEW" tag)
- **Escalated risks** that have increased in severity or likelihood
- **De-escalated risks** that have been effectively mitigated
- **Key issues** — problems that have already materialized and require board attention or action
  - For each issue: what happened, financial impact, management response, board action requested (if any)

**Formatting guidance:**

- Use the risk framework categories and severity/likelihood definitions from the firm's risk framework document
- Change column uses arrows or tags: NEW, increased risk, decreased risk, unchanged
- Issues (materialized risks) should be visually distinct from risks (potential future events)
- Keep to the top 7-10 risks. A register with 25 items is not being curated.

**Common mistakes to avoid:**

- Risk registers that never change. If the same 5 risks appear unchanged for 4 quarters, the team is not actively managing risk.
- Risks without owners. Every risk needs a named individual responsible for monitoring and mitigation.
- Confusing risks and issues. A risk is something that might happen. An issue is something that has happened. Different response required for each.
- Omitting regulatory and compliance risks. For healthcare, environmental, and financial services portfolio companies, regulatory risk is often the highest-impact category.

## Section 9: Capital Structure

**Purpose:** Report on the company's debt, leverage, covenant compliance, and liquidity position. The board needs confidence that the company can service its debt, fund its operations, and pursue growth investments.

**Target length:** 1-2 slides

**Required content:**

**Slide 9a: Debt Summary**

```
Facility          Original    Current     Rate           Maturity
                  Balance     Balance
Term Loan         $[X.X]M     $[X.X]M     SOFR+[X]bps   [Date]
Revolver ($[X]M)  ---         $[X.X]M     SOFR+[X]bps   [Date]
Other             $[X.X]M     $[X.X]M     [Rate]        [Date]
-----------       ----------  ----------
Total Debt        $[X.X]M     $[X.X]M

Cash on Hand                  $[X.X]M
Net Debt                      $[X.X]M
```

**Slide 9b: Leverage & Covenant Compliance**

| Metric | Covenant | Current | Cushion | Prior Qtr |
|--------|----------|---------|---------|-----------|
| Total Leverage (Debt/EBITDA) | [X.X]x max | [X.X]x | [X.X]x | [X.X]x |
| Fixed Charge Coverage | [X.X]x min | [X.X]x | [X.X]x | [X.X]x |
| Capex Limit | $[X.X]M max | $[X.X]M | $[X.X]M | $[X.X]M |

**Liquidity summary:**

```
Cash on Hand           $[X.X]M
Revolver Available     $[X.X]M
Less: Letters of Credit $([X.X]M)
-----------
Total Liquidity        $[X.X]M
```

- Debt amortization schedule (next 12 months and through maturity)
- Interest rate analysis (fixed vs. floating, hedging in place)
- Upcoming maturity dates or refinancing requirements
- Planned incremental borrowing (for add-on acquisitions or capex)

**Formatting guidance:**

- Leverage trend should be shown graphically (4-8 quarter bar chart)
- Covenant cushion should be highlighted — green if >20% cushion, amber if 10-20%, red if <10%
- Maturity dates approaching within 18 months should be flagged
- If interest rate hedging is in place, include the hedge details (notional, rate, expiration)

**Common mistakes to avoid:**

- Presenting leverage without the cushion analysis. The board needs to know: how much can EBITDA decline before we trip a covenant?
- Ignoring the revolver draw. A $25M revolver with $20M drawn is not "$25M of liquidity" — it is $5M of liquidity.
- Omitting interest expense trends. In a rising rate environment, floating-rate debt means the cash interest burden is growing even if the balance is declining.
- Failing to project forward. If a covenant trips at 4.5x and current leverage is 3.8x and EBITDA is flat, are we safe? What if EBITDA declines 15%?

## Section 10: Appendix

**Purpose:** Provide detailed supporting data for any board member who wants to drill deeper. The appendix should be comprehensive but clearly labeled — board members should not need to search through it to find the main-body conclusions.

**Target length:** 10-15 slides (as needed)

**Standard appendix sections:**

| Section | Contents |
|---------|----------|
| A: Detailed Financials | Full P&L, balance sheet, cash flow statement — quarterly and YTD |
| B: Revenue Detail | Revenue by customer/segment/geography — full detail |
| C: EBITDA Bridge | Reported to Adjusted EBITDA walkthrough |
| D: Working Capital | AR aging, AP aging, inventory detail, DSO/DPO/DIO trends |
| E: Capex Detail | Project-by-project capex breakdown, maintenance vs. growth |
| F: Headcount Detail | Department-level headcount, compensation by level, open positions |
| G: Customer Analysis | Top 20 customers with revenue, growth, contract expiration |
| H: Pipeline Detail | Detailed opportunity list with probability weighting |
| I: Add-on Integration | Post-acquisition integration milestones and KPIs for recent add-ons |
| J: Covenant Calculations | Detailed covenant computation worksheets |
| K: Board Approvals | Summary of actions requiring board vote or approval |

**Formatting guidance:**

- Each appendix section should be clearly labeled with a section header slide
- Cross-reference main-body slides to appendix sections (e.g., "See Appendix A for detailed financials")
- Data tables should be formatted for readability: alternating row shading, clear headers, consistent rounding
- Include the date and period for every data table

## Quarterly Reporting Calendar

Management should adhere to the following timeline for board reporting:

| Event | Timing |
|-------|--------|
| Quarter-end close | Month-end + 10 business days |
| Draft board deck to Northwood deal lead | Board meeting - 10 business days |
| Northwood feedback on draft | Board meeting - 7 business days |
| Final board deck distributed | Board meeting - 3 business days |
| Board meeting | Per schedule (typically 6-8 weeks after quarter-end) |
| Board minutes circulated | Board meeting + 5 business days |
| Action items follow-up | Board meeting + 10 business days |
